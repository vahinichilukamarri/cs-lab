1. Disk Imaging & Forensics

# STEP 1: Create evidence file
echo "Cybersecurity Lab Evidence" > evidence.txt

# STEP 2: Verify file
ls

# STEP 3: Check disks
lsblk

# STEP 4: Install dc3dd
sudo apt update
sudo apt install dc3dd -y

# STEP 5: Create 100MB practice disk
dd if=/dev/zero of=practice_disk.dd bs=1M count=100

# STEP 6: Format disk
mkfs.ext4 practice_disk.dd

# STEP 7: Create mount directory
sudo mkdir /mnt/practice_disk

# STEP 8: Mount disk
sudo mount -o loop practice_disk.dd /mnt/practice_disk

# STEP 9: Copy evidence into mounted disk
sudo cp evidence.txt /mnt/practice_disk/

# STEP 10: Verify copied file
ls /mnt/practice_disk

# STEP 11: Unmount disk
sudo umount /mnt/practice_disk

# STEP 12: Create forensic image
dc3dd if=practice_disk.dd of=disk_image.dd hash=md5 log=acquisition.log

# STEP 13: Verify image
ls -lh disk_image.dd

# STEP 14: View acquisition log
cat acquisition.log

7.Android emulator

    Procedure
Step 1: Create Android Virtual Device
Step 2: Select Device
Choose any device such as: Pixel 2 Pixel 9
Step 3: Start AVD from Command Prompt1

Open Command Prompt and execute:

cd C:\Users\Admin\AppData\Local\Android\Sdk\emulator
emulator -list-avds
Example:

Medium_Phone_API_36.1
Pixel_2
Pixel_9

Run emulator:

emulator -avd Pixel_9 -http-proxy http://10.0.2.2:8080
Step 4: Setup Proxy

In AVD:

Settings
→ Network and Internet
→ Internet
→ Select icon
→ Top right corner there is edit
→ Advanced options
→ Proxy
→ Change proxy None → Manual
→ Hostname as 10.0.2.2
→ Save
Note:

If save option is not working then follow the steps in command prompt.

Command Prompt2
cd C:\Users\Admin\AppData\Local\Android\Sdk\platform-tools
adb devices

Example:

List of devices attached
emulator-5554   device

Set proxy:

adb shell settings put global http_proxy 10.0.2.2:8080

Verify proxy:

adb shell settings get global http_proxy

Expected output:

10.0.2.2:8080
Step 5: In BurpSuite

Go:

Proxy
→ Options / Proxy Settings
→ Under Proxy Listeners
→ Edit
→ Select All interfaces
→ OK

Then:

Proxy
→ Intercept ON

You can see BurpSuite is accepting all interfaces traffic.

Step 6

In AVD type in Chrome:

example.com

You can see BurpSuite HTTP history.

Step 7

In Step 5 we only able to HTTP request and response only but if you want to analyse HTTPS request and response follow the procedure to download CA certificate from BurpSuite.

Go:

Proxy
→ Options / Proxy Settings
→ Import / Export CA Certificate
→ Certificate in DER format
→ Select file
→ Downloads
→ Give name as burpcer.der
Step 8: Sending CA Certificate to AVD

Go to Command Prompt2:

cd C:\Users\Admin\AppData\Local\Android\Sdk\platform-tools

Run:

adb push C:\Users\Admin\Downloads\burpcer.der /sdcard/Download/
To install Certificate in AVD
Settings
→ Security & privacy
→ More security & privacy
→ Encryption & credentials
→ Install a certificate
→ CA certificate

In AVD we can see:

burpcer.der

Install the certificate.

Step 9: Verify HTTPS Interception
Open browser in emulator.
Visit any HTTPS website.

Example:

https://example.com
In Burp Suite, check:
Proxy → HTTP History

Now HTTPS requests and responses will be visible.
